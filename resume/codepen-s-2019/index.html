What we built for you in 2019

We've been building CodePen for 8 years.

👋 Hey! Since 2012, we've been trying to make CodePen the most useful and fun social coding platform for front-end developers and designers as we can.

We've built loads of features in that time for both free and PRO users. Let's take a look at some of the features we built for you just in 2019.

But nothing compares to the incredible things y'all build each year, of course 😉.

You can restore your deleted Pens in case of an accident.

We trashed the place, you might say.

We don't actually call it a trash can, as CodePen is a rather international place and not all countries use that term. There are rubbish bins, waste containers, garbage pails, litter piles...

Ultimately, the tab in your Dashboard is called "Deleted" and it's where you can restore your work from if you accidently delete it.

The code for Embedded Pens can live on your own website.

We call this "Prefill Embeds".

Say you're building a documentation site. It might be easier for you to keep all your demo code in the Git repo rather than have to maintain the demos directly on CodePen. Now, you can.

Embeds are now faster and don't cause page-reflow as they load.

We overhauled the Job Board.

The Job Board got a fresh coat of paint (gradient people!) but we went deeper than that.

  • Posting a job got a lot easier. It's a single screen and you're done, whether you have an account or not.
  • Jobs stay on the board for 60 days instead of 30 at the same price.
  • Your jobs are still automatically cross-posted to CSS-Tricks and ShopTalk Show.

Specific Frameworks, Libraries, and UI patterns got homes in Topics.

Topics are templates, examples and resources for common libraries, frameworks and UI patterns.

We've got topics for the big JavaScript UI frameworks like React and Vue. We've got the popular animation libraries like Greensock and anime.js. Our very own Stephen Shaw's Splitting.js is in there. CSS heavy hitters like Sass and Tailwind are there.

Plus, we've got a Topic for all the major UI patterns on websites like Tabs, Accordions, and Dialogs.

The Pen Editor went entirely responsive.

It's not that we didn't have an editor that worked on mobile devices before. We did. But it was an entirely separate world of code that powered it. So in one sense, it's probably us who benefits from this the most since there is now one codebase powering the Pen Editor on any screen.

In another sense, the new mobile editor should be way nicer for you as well. Mostly because it's now full-featured. You can use the console, you can comment, you can analyze and tidy your code... it's a great editing experience on mobile and desktop.

We created our own high-contrast syntax highlighting themes.

Klare Frank put these together (there is both a dark and a light version) and they are, we think, hands down the nicest themes we offer. They are based somewhat on CodePen's own brand colors, which somehow work perfectly.

Managing Collections got a whole lot better.

A lot of things related to Collections got better this year. The top improvement is the whole UI/UX around adding Pens to Collections. Rather than a basic dropdown menu, you get an interface designed just for the job with recent Collections up top, searching, creating new Collections, and more. We did a few more things as well:

  • We clarified how Private Pens in Collections works.
  • Collections now offer List View
  • Collections in the grid show a 4-up preview of the Pens it contains.

Exported Pens can come with a Build Process

You've always been able to export a .zip file of the code you write. But now, PRO users can export that code, npm install, and keep working on it (even offline). The build process will process code just like CodePen does for you.

The Popout Preview

On any Pen or Project that you see in a grid (like on the homepage, your profile, search results, etc) you can click in the upper-right and expand the Pen into a full-blown preview.

From there, you can resize and interact with the preview. You can see the code. You can comment. You can see the tags and stats. All without leaving your place.

So. Much. Infastructure.

🛠

  • We totally re-wrote our search system, moving from Solr to Elasticsearch.
  • The front-end stack we've grown to be pretty happy with is React and Apollo GraphQL and now most of our front end is built that way, although we have plenty to go.
  • Ruby on Rails is our backend which we've upgraded many versions.
  • We've gotten both our own codebase and the preprocessor in the Pen Editor on Babel 7.
  • We moved all our Node code to a single monorepo, which is a big efficiency upgrade.
  • We changed the domain for all previews to a new more secure location.
  • We went to war with spam, building new tooling to catch and destroy all who seek to litter their garbage here.
  • We built internal tools to improve and quicken our dev environments.

Private by Default.

It's just a little toggle switch you turn on in your Settings and then everything you save is, you guessed it, Private by default. You can still toggle it back to public if you wish, but if most of the work you do is private, this will keep you from accidently leaving things public.
